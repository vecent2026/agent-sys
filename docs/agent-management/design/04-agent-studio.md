## 1. 功能概述

### 1.1 智能体的定义

在本平台中，**智能体（Agent）** 是一个以大语言模型为核心推理引擎，能够理解自然语言、调用外部工具、查询私有知识库、执行多步推理并完成复杂任务的自主执行单元。

智能体不同于简单的 LLM 对话接口——它封装了特定的角色身份、能力边界和行为策略，是一个可配置、可发布、可观测的运行时服务单元。

### 1.2 智能体的 5 大构成

| 构成维度 | 说明 |
|----------|------|
| 身份（Identity） | 智能体的名称、描述、头像、系统提示词，决定其角色定位和对话风格 |
| 模型（Model） | 驱动智能体的大语言模型及推理参数配置，决定智能体的推理能力上限 |
| 工具（Tools） | 智能体可调用的外部能力集合（HTTP API / MCP 工具 / 代码执行等），扩展智能体的行动边界 |
| 知识（Knowledge） | 关联的知识库集合，为智能体提供私有领域知识的实时检索能力 |
| 行为策略（Behavior） | 执行范式、循环步数上限、超时策略、安全限制，控制智能体的运行边界和资源消耗 |

### 1.3 智能体类型

| 类型 | 说明 | 典型使用场景 |
|------|------|-------------|
| 对话型 Agent（Chat Agent） | 以多轮对话为交互形式，理解上下文并给出连贯回答，工具调用是可选增强 | 客服机器人、问答助手、知识检索助手 |
| 工具调用型 Agent（Tool-use Agent） | 以完成具体任务为目标，主动决策调用哪些工具，工具调用是核心能力 | 数据查询、报表生成、系统操作自动化 |
| ReAct Agent | 采用「推理（Reasoning）→ 行动（Action）→ 观察（Observation）」循环，支持复杂多步任务分解 | 复杂业务流程自动化、多系统协同任务 |

**各类型推荐配置默认值**

| Agent 类型 | 执行范式 | temperature 默认值 | max_steps 默认值 | supports_tool_call 要求 |
|-----------|---------|-------------------|-----------------|------------------------|
| 对话型 | Single-turn Chat | 0.7 | 1 | 不要求 |
| 工具调用型 | Tool-Use Loop | 0.3 | 5 | 必须 |
| ReAct | ReAct (Reason+Act) | 0.5 | 10 | 必须 |

### 1.4 与 Skill、Workflow 的区别

#### 概念定义

- **Skill（技能）**：一段封装好的、可复用的 LLM 调用逻辑，输入固定、输出固定，无自主决策能力。类似函数调用。
- **Workflow（工作流）**：由多个 Skill / 工具节点组成的有向流程图，执行路径在设计时确定（非 LLM 动态决策）。
- **Agent（智能体）**：具备自主推理和决策能力，能够根据当前状态动态选择下一步行动，适用于目标明确但路径不固定的任务。

#### 决策矩阵

| 场景特征 | 推荐方案 | 理由 |
|----------|----------|------|
| 任务路径固定，输入输出明确 | Skill | 轻量、可预期、易调试 |
| 多步骤、有分支、但分支逻辑可枚举 | Workflow | 流程可视化、行为确定性高 |
| 需要多轮对话理解用户意图 | Agent（对话型） | 语境跟踪、自然语言理解 |
| 需要调用多个工具，路径由 LLM 动态决策 | Agent（工具调用型 / ReAct） | 自主决策、灵活应对未知路径 |
| 涉及知识库检索 + 工具调用组合 | Agent | 知识与行动一体化 |
| 高并发、低延迟、无状态 | Skill / Workflow | Agent 推理开销相对较大 |

## 2. 智能体列表页

### 2.1 展示方式

列表页默认使用**卡片网格**布局（推荐，视觉直观，便于展示智能体头像和描述摘要），同时提供表格视图切换按钮（适合管理大量智能体的场景）。

**卡片网格规则**：
- 每行 4 卡片（桌面 1440px+）/ 3 卡片（1280px）/ 2 卡片（768px）
- 每个卡片高度固定 180px，悬浮时显示阴影增强（`box-shadow: 0 8px 16px rgba(15, 23, 42, 0.12)`）

**每张卡片展示内容**：

| 元素 | 说明 |
|------|------|
| 头像 | 圆形头像（48×48），若未上传则显示首字母彩色背景 |
| 名称 | 智能体名称，最多显示 2 行，超出省略 |
| 状态 Badge | 草稿（灰色）/ 已发布（绿色）/ 已归档（黄色） |
| 模型标签 | 当前配置的主模型名称（小标签样式） |
| 描述 | 最多显示 2 行，超出省略 |
| 最近更新时间 | 相对时间（如「3 天前」），悬浮显示绝对时间 |
| 操作菜单 | 卡片右上角三点菜单：进入配置 / 查看观测 / 复制 / 删除 |

### 2.2 搜索与筛选

工具栏筛选项（变更后立即触发查询，无需点击查询按钮）：

| 筛选项 | 控件 | 说明 |
|--------|------|------|
| 名称搜索 | 搜索输入框（防抖 300ms） | 按名称模糊匹配 |
| 状态 | Select 下拉 | 全部 / 草稿 / 已发布 / 已归档 |
| 标签 | 多选 Select | 按智能体标签过滤，支持多选 |
| 创建人 | Select（仅管理员可见） | 按创建人过滤 |

### 2.3 操作说明

| 操作 | 说明 |
|------|------|
| 进入配置 | 跳转到智能体配置页（Agent Studio） |
| 查看观测 | 跳转到该智能体的 Trace 观测面板，查看历史调用记录 |
| 复制 | 复制智能体配置，创建一个名称为「{原名称} 的副本」的草稿 |
| 删除 | 已发布的智能体需先归档才能删除；草稿和已归档可直接删除（二次确认） |

> **复制说明**：复制时完整克隆 `draft_config`，包括所有已绑定 Skill 的版本锁定（若绑定了指定版本，副本也保持同一版本，不自动升级为 latest），确保副本行为与原 Agent 一致。

### 2.4 新建智能体流程

点击工具栏「新建智能体」按钮（primary 样式），右侧 Drawer 打开，包含最简表单（快速创建草稿，避免在空配置页面迷失）：

| 字段 | 类型 | 是否必填 | 说明 |
|------|------|----------|------|
| 名称 | string | 是 | 最长 64 字符，同租户内唯一 |
| 描述 | string | 否 | 最长 256 字符 |
| 智能体类型 | Radio | 是 | 对话型 / 工具调用型 / ReAct，选择后影响配置页默认参数 |

点击「创建并进入配置」，创建草稿并立即跳转至配置页。

## 3. 智能体配置页面布局

### 3.1 整体布局

配置页采用**双栏布局**，不带侧边导航（避免视觉干扰，让用户专注于配置和调试）：

```
┌─────────────────────────────────────────────────────────────────────┐
│  顶部操作栏（高度 56px）                                               │
├──────────────────────────────┬──────────────────────────────────────┤
│                              │                                      │
│   左侧配置区（40% 宽）         │   右侧调试对话区（60% 宽）              │
│                              │                                      │
│   折叠分区列表               │   消息列表（可滚动）                    │
│   ├─ 基础信息                │                                      │
│   ├─ 提示词配置              │                                      │
│   ├─ 模型与推理参数          │   工具调用展示 / 推理步骤展示           │
│   ├─ 工具配置                │                                      │
│   ├─ 知识库配置              │   底部输入区                          │
│   ├─ Skill 配置 [P1]         │   （输入框 + 发送按钮 + 清空对话）     │
│   ├─ 记忆配置 [P1]           │                                      │
│   ├─ 行为策略                │                                      │
│   └─ 安全与限制              │                                      │
│                              │                                      │
└──────────────────────────────┴──────────────────────────────────────┘
```

左右两栏宽度比例可通过拖动分隔线调整（范围 30%-60% 左侧）。

### 3.2 顶部操作栏

| 区域 | 元素 | 说明 |
|------|------|------|
| 左侧 | 返回按钮 + 智能体名称（可内联编辑） | 点击名称直接编辑，失焦保存 |
| 中间 | 保存状态提示 | 三种状态：「已保存」（绿色文字）/ 「保存中…」（灰色 + loading icon）/ 「有未保存的修改」（橙色文字） |
| 右侧 | 状态 Badge + 版本历史按钮 + 发布按钮 | 草稿状态下「发布」为 primary 样式；已发布状态下显示「更新发布」；点击「版本历史」弹出历史版本列表 Drawer |

**版本历史 Drawer**：展示历史发布记录（版本号 / 发布时间 / 发布人 / 变更说明），支持回滚到任意历史版本（回滚后创建新草稿，不直接覆盖当前发布版本）。

### 3.3 左侧配置区折叠分区结构

每个配置分区为可折叠的手风琴（Collapse）组件，默认全部展开。

- 分区标题栏：左侧分区名称 + 右侧折叠/展开 icon
- 区与区之间有 1px 分隔线
- 折叠状态保存到 localStorage，刷新页面后保持上次状态

### 3.4 右侧调试对话区

**组件结构**：

```
┌────────────────────────────────────┐
│ 调试对话区标题 + 「清空对话」按钮    │
├────────────────────────────────────┤
│                                    │
│  消息列表（可滚动，用户消息右对齐，  │
│  Agent 消息左对齐）                  │
│                                    │
│  ┌──────────────────────────────┐   │
│  │ 工具调用展示（可折叠）         │   │
│  │  ├─ 工具名称 + 调用参数       │   │
│  │  └─ 工具返回结果              │   │
│  └──────────────────────────────┘   │
│                                    │
│  ┌──────────────────────────────┐   │
│  │ 推理步骤展示（仅 ReAct/P&E）  │   │
│  │  ├─ Thought: 推理过程         │   │
│  │  ├─ Action: 选择工具          │   │
│  │  └─ Observation: 工具结果     │   │
│  └──────────────────────────────┘   │
│                                    │
├────────────────────────────────────┤
│ 输入框（多行，Shift+Enter 换行）     │
│ [附件] [清空对话]       [发送]       │
└────────────────────────────────────┘
```

**调试区特性**：
- 调试对话使用当前**草稿配置**执行（不需要发布）
- 调试模式下 Token 消耗计入该智能体的测试用量，不计入生产用量
- 每次修改配置后，调试区显示「配置已更新，下次对话将使用最新配置」提示横条
- 支持展开「详细信息」查看：每轮对话的 Prompt Token 数、Completion Token 数、耗时

**Trace 采集策略**：所有对话调用（包括调试模式）均全量采集 Trace，但调试模式的 Trace 打上 `namespace=debug` 标签，发布后的生产调用打上 `namespace=production`。调试 Trace 在可观测性面板中默认隐藏，需勾选「包含调试记录」才显示。这样既能追溯问题，又不污染生产指标。

## 4. 完整配置维度

### 4.1 基础信息

| 字段名 | 类型 | 是否必填 | 默认值 | 说明 |
|--------|------|----------|--------|------|
| name | string | 是 | — | 智能体名称，最长 64 字符，同租户内唯一 |
| description | string | 否 | — | 智能体描述，最长 256 字符，对调用方可见 |
| avatar | string / emoji | 否 | 随机颜色首字母 | 上传图片（128×128 PNG/JPG，最大 200KB）或选择 Emoji |
| tags | string[] | 否 | [] | 标签列表，每个标签最长 20 字符，最多 10 个标签，用于分类筛选 |
| opening_message | string | 否 | — | 开场白：智能体首次被调用时（对话历史为空）自动发送的欢迎消息，最长 500 字符 |
| suggested_questions | string[] | 否 | [] | 建议问题列表，最多 5 条，每条最长 100 字符；开场白展示后，以快捷按钮形式显示，用户点击即发送 |

**开场白与建议问题的展示时机**：仅在对话历史为空（新会话开始时）触发。开场白作为 Assistant 角色的第一条消息展示，建议问题以标签形式展示在输入框上方。

### 4.2 提示词配置

#### 系统提示词编辑器

提供富文本编辑区域（等宽字体，行号显示，高亮 `{{variable}}` 变量语法）：

| 字段名 | 类型 | 是否必填 | 默认值 | 说明 |
|--------|------|----------|--------|------|
| system_prompt | string | 否 | — | 系统提示词正文，支持 `{{变量名}}` 插值语法，最长 32000 字符 |

编辑器右下角实时显示：字符数 / Token 估算数（使用 tiktoken cl100k_base 估算，注明「仅估算，实际以模型计费为准」）。

**提示词最佳实践建议**（以折叠提示区展示在编辑器下方）：
- 清晰定义角色和职责范围
- 明确输出格式要求
- 设置必要的行为约束

#### 变量插值语法

系统提示词支持 `{{变量名}}` 语法引用变量，运行时自动替换。变量分两类：

**内置变量**（平台自动注入，无需配置）：

| 变量名 | 类型 | 说明 |
|--------|------|------|
| `{{current_date}}` | string | 当前日期，格式 YYYY-MM-DD |
| `{{current_datetime}}` | string | 当前日期时间，格式 YYYY-MM-DD HH:mm:ss |
| `{{tenant_name}}` | string | 当前租户名称 |
| `{{user_name}}` | string | 当前调用用户的显示名称（API 调用时为 API Key 关联用户名） |
| `{{user_id}}` | string | 当前调用用户的 ID |
| `{{context}}` | string | 知识库检索结果注入点（仅在注入方式为「System Prompt 末尾」时有效；未设置此变量时，知识库内容追加到 System Prompt 末尾） |

**自定义变量**（智能体配置者定义，调用方通过 API 请求体传入）：

| 字段名 | 类型 | 是否必填 | 默认值 | 说明 |
|--------|------|----------|--------|------|
| key | string | 是 | — | 变量名，仅允许字母/数字/下划线，不可与内置变量同名 |
| type | enum | 是 | string | 变量类型：string / number / boolean |
| default_value | any | 否 | — | 默认值，调用方未传入时使用此值 |
| required | boolean | 是 | false | 是否必须由调用方传入（true 时调用方不传入则返回 400） |
| description | string | 否 | — | 变量说明，显示在 API 文档中，帮助调用方理解含义 |

自定义变量列表以表格形式展示，支持内联增删改。

#### 输出格式配置

| 字段名 | 类型 | 是否必填 | 默认值 | 说明 |
|--------|------|----------|--------|------|
| output_format | enum | 是 | text | 输出格式：text（纯文本）/ structured（结构化 JSON） |
| output_schema | object | 否（structured 时必填） | — | JSON Schema，定义结构化输出的字段和类型 |

**STRUCTURED 模式的 JSON Schema 编辑器**：

提供两种编辑模式（Tab 切换）：
1. **可视化编辑**：以树状列表展示，支持添加字段（字段名 / 类型：string/number/boolean/array/object / 是否必填 / 描述）
2. **原始 JSON**：Monaco Editor 风格的 JSON 编辑器，实时语法验证

示例（用于结构化信息提取场景）：
```json
{
  "type": "object",
  "properties": {
    "summary": { "type": "string", "description": "摘要" },
    "sentiment": { "type": "string", "enum": ["positive", "neutral", "negative"] },
    "key_points": { "type": "array", "items": { "type": "string" } }
  },
  "required": ["summary", "sentiment"]
}
```

### 4.3 模型与推理参数

#### 主模型选择

| 字段名 | 类型 | 是否必填 | 默认值 | 说明 |
|--------|------|----------|--------|------|
| model_id | string | 是 | 租户默认模型 | 选择主模型，下拉框仅展示当前租户已启用的模型 |

下拉选项展示格式：模型名称 + 能力标签（如「GPT-4o」+ [视觉] [128K] [工具调用]）。标签含义：
- [视觉]：支持图片输入
- [128K] / [200K] 等：上下文窗口大小
- [工具调用]：支持 Function Calling / Tool Use
- [推理]：原生推理模型（如 o1/o3 系列，支持 thinking_budget）
- [流式]：支持流式输出

#### 推理参数

| 参数名 | 类型 | 是否必填 | 默认值 | 范围 | 说明 |
|--------|------|----------|--------|------|------|
| temperature | float | 是 | 0.7 | 0.0 - 2.0 | 生成多样性：0 = 完全确定性（适合事实问答），2 = 极高随机性（适合创意写作）；以滑块控件展示，同时支持直接输入数字 |
| top_p | float | 是 | 0.9 | 0.0 - 1.0 | 核采样概率阈值：只从累积概率超过 top_p 的 Token 中采样；通常与 temperature 任选其一调整，不建议同时改动两者 |
| max_tokens | integer | 是 | 2048 | 1 - 模型最大值 | 单次响应的最大 Token 数；上限由所选模型决定（系统自动显示该模型的最大允许值） |
| thinking_budget | integer | 否（仅推理模型显示） | 0 | 0 - 32000 | 推理 Token 预算：0 = 关闭推理，正整数 = 允许使用的最大推理 Token 数；仅在选择推理模型时显示此字段 |
| presence_penalty | float | 否 | 0.0 | -2.0 - 2.0 | 存在惩罚：正值降低重复出现 Token 的概率（高级选项，默认折叠） |
| frequency_penalty | float | 否 | 0.0 | -2.0 - 2.0 | 频率惩罚：正值降低高频出现 Token 的概率（高级选项，默认折叠） |

高级选项（presence_penalty / frequency_penalty）默认折叠，点击「高级参数」展开。

#### 备用模型配置

| 字段名 | 类型 | 是否必填 | 默认值 | 说明 |
|--------|------|----------|--------|------|
| fallback_model_id | string | 否 | — | 备用模型：主模型返回错误（5xx）或超时时自动切换，选填 |
| fallback_on_timeout | boolean | 否 | true | 超时时是否触发备用模型 |
| fallback_on_error | boolean | 否 | true | 主模型返回 5xx 错误时是否触发备用模型 |

#### 上下文窗口策略

## 5. 智能体状态机

### 5.1 状态定义

| 状态 | 标识符 | Badge 颜色 | 描述 |
|------|--------|------------|------|
| 草稿 | DRAFT | 灰色 | 初始状态，配置未发布，只能在调试区测试，不可通过 API 调用 |
| 已发布 | PUBLISHED | 绿色 | 最新配置已发布，可通过 API 调用；同时可继续修改草稿（修改不影响已发布版本） |
| 已归档 | ARCHIVED | 黄色/橙色 | 已停止服务，不可通过 API 调用；配置可查看但不可修改；可恢复为草稿状态 |

### 5.2 状态流转图

```
                    ┌─────────────────────────────────────────┐
                    │                                         │
         创建        │          发布                           │  取消发布
  ──────► DRAFT ────┼──────────────► PUBLISHED               │
                    │                    │                   │
                    │                    │ 归档               │
                    │                    ▼                   │
                    │               ARCHIVED ────────────────┘
                    │                    │
                    │                    │ 恢复草稿
                    └────────────────────┘
                    （ARCHIVED → DRAFT，创建新草稿）
```

详细转换规则：

| 起始状态 | 目标状态 | 触发动作 | 条件 |
|----------|----------|----------|------|
| DRAFT | PUBLISHED | 点击「发布」 | 名称不能为空；至少配置了主模型 |
| PUBLISHED | DRAFT（草稿并存） | 修改任意配置后自动创建草稿 | 草稿与已发布版本并存 |
| PUBLISHED | ARCHIVED | 点击「归档」 | 二次确认，归档后 API 调用立即失败 |
| ARCHIVED | DRAFT | 点击「恢复为草稿」 | 创建新草稿，历史发布记录保留 |
| DRAFT | ARCHIVED | 不支持 | — |

### 5.3 各状态下允许的操作

| 操作 | DRAFT | PUBLISHED | ARCHIVED |
|------|-------|-----------|----------|
| 修改配置 | 允许 | 允许（修改后进入并存草稿状态） | 禁止 |
| 发布 | 允许 | 允许（更新发布） | 禁止 |
| 调试对话 | 允许 | 允许（使用草稿配置） | 禁止 |
| API 调用 | 禁止 | 允许 | 禁止 |
| 查看观测数据 | 允许（仅调试数据） | 允许 | 允许 |
| 归档 | 禁止 | 允许 | — |
| 恢复草稿 | — | — | 允许 |
| 删除 | 允许 | 禁止（须先归档） | 允许 |
| 查看版本历史 | 允许 | 允许 | 允许 |

## 6. 草稿自动保存

### 6.1 保存策略

配置页采用 **debounce 自动保存**策略，无需用户手动点击「保存」：

- 用户修改任意配置项后，启动 800ms debounce 计时器
- 800ms 内无新修改，触发异步保存请求（PATCH `/api/v1/agents/{agent_id}/draft`）
- 保存期间不阻塞 UI，用户可继续修改
- 若保存请求失败（网络错误等），顶部显示错误提示横条，5 秒后消失，允许手动重试

**手动保存**：顶部操作栏提供「保存」快捷按钮（仅在「有未保存的修改」状态时高亮显示），以及 Ctrl/Cmd + S 快捷键触发立即保存。

### 6.2 保存状态提示

顶部操作栏中间区域实时显示保存状态：

| 状态 | 显示内容 | 样式 |
|------|----------|------|
| 已保存 | 「已保存」 + 最后保存时间（相对时间） | 绿色文字 + 对勾图标 |
| 保存中 | 「保存中…」 | 灰色文字 + Loading Spinner |
| 有未保存的修改 | 「有未保存的修改」 | 橙色文字 + 感叹号图标 |
| 保存失败 | 「保存失败，点击重试」 | 红色文字 + 重试图标，可点击触发重试 |

### 6.3 草稿与已发布版本并存

核心设计原则：**修改草稿不影响已发布版本的 API 服务**。

实现机制：
- 数据库中每个 Agent 同时存储两份完整配置：`draft_config`（草稿配置 JSON）和 `published_config`（已发布配置 JSON）
- API 调用始终使用 `published_config`
- 调试对话始终使用 `draft_config`
- 发布动作：将 `draft_config` 覆盖写入 `published_config`，生成一条版本历史记录
- 版本历史存储：每次发布时对 `published_config` 打快照，保留最近 20 个发布版本

## 7. 接口说明概要
