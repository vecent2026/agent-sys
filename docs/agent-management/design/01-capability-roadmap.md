## 3. 各能力域详细清单

### 3.1 大模型接入管理（LLM Gateway）

**功能目标**：统一管理大模型供应商接入，为平台所有智能体提供稳定、安全、可计量的 LLM 调用能力。

#### 3.1.1 供应商接入

| 功能     | 描述                                                                                                                             |
| ------ | ------------------------------------------------------------------------------------------------------------------------------ |
| 多供应商支持 | OpenAI / Azure OpenAI / Anthropic Claude / Google Gemini / Moonshot / 通义千问 / 百川 / 文心一言 / DeepSeek / Ollama（本地）/ vLLM（本地）/ 自建网关 |
| 供应商适配器 | 每个供应商对应独立适配器，处理接口差异（认证方式/请求格式/流式协议）                                                                                            |
| 自建网关兼容 | 支持 OpenAI 兼容协议的自建网关（Base URL 自定义）                                                                                              |
| 模型预置模板 | 平台内置主流模型的配置模板（OpenAI GPT-4o / Claude Sonnet / Gemini Flash 等），新建时一键预填                                                          |

#### 3.1.2 模型配置管理

| 字段            | 说明                             |
| ------------- | ------------------------------ |
| 模型名称          | 管理展示用名称（如 "GPT-4o 正式环境"）       |
| 模型标识          | 调用时的 model 字段（如 `gpt-4o`）      |
| 供应商类型         | 枚举值，决定使用哪个适配器                  |
| Base URL      | 基础请求地址                         |
| API 路径 / 接口类型 | `/v1/chat/completions` 或接口类型枚举 |
| API 版本        | Azure 等需要 api-version 参数       |
| 密钥配置 ID       | 关联 Secret Manager 中的密钥         |
| 请求超时（ms）      | 单次调用最大等待时间                     |
| 启用状态          | 启用 / 停用（停用后租户无法选用）             |
| 是否默认模型        | 全局唯一，新建智能体时预选                  |
| 模型能力标注        | 见下方详细说明                        |
| 最近测试状态        | 可用 / 存在问题 / 未测试                |
| 最近测试时间        | 最后一次可用性测试的时间戳                  |
| 最近测试消息        | 测试失败时的错误信息                     |

**模型能力标注（供智能体配置时参考）：**

| 能力标注字段                    | 说明                              | 示例值          |
| ------------------------- | ------------------------------- | ------------ |
| 支持工具调用（Function Calling）  | 是否支持 tool\_use / function\_call | true / false |
| 支持结构化输出（JSON Mode）        | 是否支持强制 JSON 输出                  | true / false |
| 支持视觉输入（Vision）            | 是否支持图片输入                        | true / false |
| 支持音频输入（Audio）             | 是否支持音频输入                        | true / false |
| 推理模型（Reasoning）           | 是否为 o1/o3/DeepSeek-R1 类推理模型     | true / false |
| 上下文窗口大小（Context Window）   | Token 数量，如 128000               | 整数           |
| 最大输出长度（Max Output Tokens） | 单次最大输出 Token 数                  | 整数           |
| 输入 Token 单价（USD/1K）       | 计费用途                            | 浮点数          |
| 输出 Token 单价（USD/1K）       | 计费用途                            | 浮点数          |
| 支持流式响应（Streaming）         | 是否支持 SSE 流式                     | true / false |

#### 3.1.3 多 API Key 管理

| 功能       | 描述                                             |
| -------- | ---------------------------------------------- |
| Key 池化   | 同一模型配置可关联多个 API Key，形成密钥池                      |
| 轮转策略     | 支持轮询（Round-Robin）/ 随机 / 最少使用（Least Used）三种轮转策略 |
| Key 健康状态 | 自动标记失效/限额的 Key，路由时跳过                           |
| 负载均衡     | 按 Key 设置权重，分摊高并发请求                             |
| Key 单独测试 | 支持对池中的每个 Key 独立执行可用性测试                         |

#### 3.1.4 可用性监控

| 功能    | 描述                                  |
| ----- | ----------------------------------- |
| 一键测试  | 手动触发，发送固定测试 Prompt，记录成功/失败、耗时、错误信息  |
| 定时拨测  | 可配置的定时任务（如每 5 分钟），自动测试并更新可用性状态      |
| 异常告警  | 测试失败时发送告警通知（邮件/站内信/Webhook 可扩展）     |
| 熔断机制  | 连续失败 N 次后自动将模型标记为「熔断」状态，停止路由（可自动恢复） |
| 可用性历史 | 记录历次测试结果，可查看可用性趋势                   |

#### 3.1.5 模型可见性控制

| 功能    | 描述                 |
| ----- | ------------------ |
| 全局可见  | 所有租户均可选用           |
| 租户白名单 | 仅指定租户可见（按租户 ID 配置） |
| 隐藏    | 仅平台管理员可见，不对租户开放    |

### 3.2 密钥与凭据管理（Secret Manager）

**功能目标**：安全存储和管理 LLM API Key 及其他凭据，API Key 明文永不暴露在日志或响应中。

| 功能              | 描述                                                   |
| --------------- | ---------------------------------------------------- |
| 加密存储            | AES-256-GCM 加密，Master Key 通过环境变量注入                   |
| 凭据类型            | API Key（当前）/ OAuth Token（扩展）/ 自定义 Header（扩展）         |
| 创建凭据            | 输入明文 Key → 加密存储 → 返回 secretConfigId（不返回明文）           |
| 密钥轮换            | 更新凭据时自动清除关联缓存，新 Key 生效无停机                            |
| 凭据引用            | 模型配置通过 secretConfigId 引用，运行时动态解密                     |
| 访问日志            | 记录每次解密访问（时间、调用方、用途）                                  |
| 外部 Vault 对接（预留） | 抽象接口 `SecretService`，支持后续迁移到 HashiCorp Vault / 云 KMS |

### 3.3 工具生态（Tool Ecosystem）

**功能目标**：为智能体提供与外部世界交互的能力，遵循 MCP 标准构建开放工具生态。

#### 3.3.1 内置工具（平台提供）

| 工具名称                   | 能力描述                               | 参数                         |
| ---------------------- | ---------------------------------- | -------------------------- |
| 网络搜索（Web Search）       | 搜索引擎检索，返回 Top-N 结果摘要               | query, num\_results        |
| 代码执行（Code Interpreter） | 在沙箱中执行 Python/JS 代码，返回输出           | code, language             |
| HTTP 请求（HTTP Client）   | 发起任意 HTTP 请求（GET/POST/PUT 等）       | url, method, headers, body |
| 文件读取（File Reader）      | 解析 PDF / Word / Excel / CSV，提取文本内容 | file\_url, type            |
| 数学计算（Calculator）       | 精确数学计算（避免 LLM 计算误差）                | expression                 |
| 日期时间（DateTime）         | 获取当前时间、日期格式转换、时区计算                 | timezone, format           |
| 数据库查询（DB Query）        | 执行只读 SQL 查询（连接租户配置的数据源）            | sql, datasource\_id        |

#### 3.3.2 MCP Server 接入

MCP（Model Context Protocol）已成为工具接入的事实标准（97M+/月下载量），平台将 MCP 作为工具接入的核心协议。

| 功能             | 描述                                            |
| -------------- | --------------------------------------------- |
| HTTP Server 模式 | 接入远程 MCP Server（HTTPS Endpoint + Bearer Auth） |
| stdio 模式       | 接入本地进程 MCP Server（适用于本地部署场景）                  |
| MCP Server 注册  | 录入 MCP Server 地址、认证信息、工具描述；平台自动发现可用工具列表       |
| 工具同步           | 自动同步 MCP Server 暴露的工具 Schema（名称、描述、参数）        |
| 工具调用代理         | 智能体调用工具时，平台代理转发请求到对应 MCP Server               |
| 可用性测试          | 对已接入的 MCP Server 执行连通性测试                      |

#### 3.3.3 自定义工具（OpenAPI 定义）

| 功能                | 描述                                                 |
| ----------------- | -------------------------------------------------- |
| OpenAPI Schema 导入 | 粘贴或上传 OpenAPI 3.0 YAML/JSON，平台自动解析工具列表             |
| 工具参数映射            | 将 API 参数映射为工具的 input schema（LLM 可读的参数描述）           |
| 鉴权配置              | 支持 API Key / Bearer Token / Basic Auth / OAuth 2.0 |
| 响应处理              | 配置如何从 API 响应中提取有用内容（JSONPath 映射）                   |

#### 3.3.4 工具市场

| 功能      | 描述                           |
| ------- | ---------------------------- |
| 平台公开工具  | 平台管理员发布，所有租户均可使用             |
| 租户私有工具  | 租户自建，仅本租户可见                  |
| 工具分类与搜索 | 按类别（搜索/数据/生产力/开发/...）+ 关键词搜索 |
| 工具使用统计  | 调用量、成功率、延迟统计                 |
| 工具上下架   | 平台管理员可对工具进行上下架管理             |

### 3.4 知识库与 RAG（Knowledge Base）

**功能目标**：让智能体能够检索和利用特定领域知识，实现知识增强的问答和分析能力。

#### 3.4.1 知识库管理

| 功能    | 描述                                 |
| ----- | ---------------------------------- |
| 创建知识库 | 设置名称、描述、Embedding 模型、默认检索策略        |
| 知识库列表 | 展示本租户所有知识库（名称/文档数/存储大小/更新时间/状态）    |
| 知识库设置 | 修改 Embedding 模型（触发重新向量化）、检索参数      |
| 知识库共享 | 租户内成员可查看/使用；管理员可设置公开给所有租户（平台公共知识库） |
| 删除知识库 | 检查是否有智能体引用，引用中时提示并阻止               |
| 知识库版本 | 记录重要变更（新增/删除大批量文档时打快照）             |

#### 3.4.2 文档管理

| 功能        | 描述                                                           |
| --------- | ------------------------------------------------------------ |
| 文件上传      | PDF / Word（.docx）/ Excel（.xlsx）/ Markdown / TXT / CSV / HTML |
| URL 爬取    | 输入 URL，自动爬取网页内容（支持递归爬取指定深度）                                  |
| API 数据源   | 配置定时从 REST API 拉取数据（支持分页，JSON 内容提取）                          |
| 数据库同步（扩展） | 连接 MySQL/PostgreSQL，将查询结果同步为文档                               |
| 文档状态      | 处理中 / 已完成 / 处理失败（含失败原因）                                      |
| 文档预览      | 查看原文内容和切分后的 Chunk 列表                                         |
| 文档更新      | 重新上传覆盖，自动触发重新处理和向量化                                          |
| 批量删除      | 支持批量选择删除                                                     |

#### 3.4.3 文档处理管道

```
[原始文档]
    │
    ▼ 解析（Parsing）
[纯文本 + 结构信息]   ← PDF / Word / HTML 各自适配解析器
    │                   多模态：图片、表格单独提取
    ▼ 清洗（Cleaning）
[清洗后文本]           ← 去除噪声（页眉页脚/广告/特殊字符）
    │
    ▼ 切分（Chunking）
[Chunks 列表]          ← 可配置切分策略（见下表）
    │
    ▼ Embedding
[向量 + 元数据]        ← 可选 Embedding 模型（见下表）
    │
    ▼ 存储
[向量数据库 + 元数据 DB]
```

**切分策略选项：**

| 策略             | 描述                                        | 适用场景             |
| -------------- | ----------------------------------------- | ---------------- |
| 固定大小切分         | 按 Token 数固定切分（如 512 Tokens），支持重叠（Overlap） | 通用场景，快速          |
| 按段落切分          | 以自然段落为边界切分                                | 文章、报告类文档         |
| 按标题切分          | 识别 Markdown / Word 标题层级，按章节切分             | 有明确章节的技术文档       |
| 语义切分（Semantic） | 使用语义相似度判断切分边界                             | 高质量需求，适合长文本      |
| 自定义分隔符         | 指定分隔符（如 `---`、特定标题）                       | 结构化的 Markdown 文档 |

**Embedding 模型选项：**

| 模型                     | 提供方         | 维度   | 适合语言    |
| ---------------------- | ----------- | ---- | ------- |
| text-embedding-3-small | OpenAI      | 1536 | 英文为主    |
| text-embedding-3-large | OpenAI      | 3072 | 英文为主    |
| text-embedding-ada-002 | OpenAI      | 1536 | 通用      |
| BGE-M3                 | 智源          | 1024 | 中英双语，推荐 |
| M3E-base               | HuggingFace | 768  | 中文为主    |
| 自定义 Embedding API      | 用户配置        | 可变   | 自建      |

#### 3.4.4 检索策略

| 策略           | 描述                                            | 优缺点             |
| ------------ | --------------------------------------------- | --------------- |
| 向量语义检索       | 将 Query Embedding 后计算与 Chunks 的余弦相似度，返回 Top-K | 语义理解强，但关键词精确匹配弱 |
| BM25 关键词检索   | 基于 TF-IDF 的传统关键词检索                            | 关键词精确匹配强，语义理解弱  |
| 混合检索（Hybrid） | 向量检索 + BM25 结果融合（RRF 或加权平均）                   | 综合两者优势，推荐默认配置   |
| Reranker 精排  | 用 Cross-Encoder 模型对 Top-K 结果重新排序              | 检索质量最高，延迟略增     |

**检索参数：**

- `top_k`：返回的 Chunk 数量（1-20，默认 5）
- `similarity_threshold`：最低相似度阈值（低于此值的 Chunk 过滤掉）
- `reranker_model`：Reranker 模型选择（可选）
- `search_method`：检索策略选择

#### 3.4.5 Agentic RAG

传统 RAG 是单步检索，Agentic RAG 让 Agent 主动决策检索过程：

| 阶段   | 功能                               |
| ---- | -------------------------------- |
| 检索决策 | Agent 判断是否需要检索（简单问候不检索，专业问题才检索）  |
| 查询改写 | Agent 根据对话历史改写检索 Query（解决指代消解问题） |
| 多轮检索 | 首次检索不满意时，Agent 自主调整 Query 执行二次检索 |
| 结果评估 | Agent 评估检索结果的相关性，决定是否采用          |
| 引用标注 | 响应中标注信息来源（知识库名称 + 文档片段）          |

#### 3.4.6 知识库评估

| 功能     | 描述                                |
| ------ | --------------------------------- |
| 检索测试   | 输入 Query，查看检索结果，评估相关性             |
| 命中率统计  | 历史查询的检索命中率分布                      |
| 相关性评分  | LLM-as-Judge 自动评估检索结果与 Query 的相关性 |
| 问答质量评估 | 基于知识库的问答准确性测试（需配置测试集）             |

### 3.5 提示词工程（Prompt Engineering）

**功能目标**：提供标准化的提示词管理能力，支持模板化复用，避免提示词分散在各智能体中难以维护。

#### 3.5.1 系统提示词

| 功能     | 描述                           |
| ------ | ---------------------------- |
| 富文本编辑器 | 支持 Markdown 格式编写提示词          |
| 变量插值   | Jinja2 风格（`{{变量名}}`），运行时动态替换 |
| 变量预览   | 填入变量值后实时预览渲染效果               |
| 字数统计   | 实时显示 Token 数量估算，防止超出上下文窗口    |

**内置变量（系统自动注入）：**

| 变量                     | 说明                  |
| ---------------------- | ------------------- |
| `{{current_date}}`     | 当前日期（YYYY-MM-DD）    |
| `{{current_datetime}}` | 当前日期时间（UTC+8）       |
| `{{tenant_name}}`      | 当前租户名称              |
| `{{user_name}}`        | 当前用户昵称              |
| `{{agent_name}}`       | 当前智能体名称             |
| `{{context}}`          | 知识库检索结果（由 RAG 模块注入） |

**自定义变量：**

- 在智能体配置中定义变量名 + 类型（字符串/数字/布尔）
- 调用方通过 API 传入变量值，实现运行时定制化

#### 3.5.2 提示词模板库

| 功能     | 描述                              |
| ------ | ------------------------------- |
| 平台公共模板 | 平台管理员创建，所有租户可用（如：客服助手模板、代码助手模板） |
| 租户私有模板 | 租户内创建，仅本租户可见                    |
| 模板分类   | 按场景分类（客服/代码/分析/创作/教育/...）       |
| 从模板创建  | 选择模板后一键套用到智能体，可继续编辑             |
| 模板收藏   | 租户收藏常用模板                        |

#### 3.5.3 提示词版本管理

| 功能   | 描述                  |
| ---- | ------------------- |
| 独立版本 | 提示词可有独立版本号，与智能体版本解耦 |
| 版本对比 | 并排查看不同版本的差异         |
| 版本切换 | 智能体可指定引用某个提示词版本     |
| 变更历史 | 查看每次变更的内容、时间、操作人    |

#### 3.5.4 AI 辅助提示词优化

| 功能   | 描述                                        |
| ---- | ----------------------------------------- |
| 优化建议 | 选中提示词后，由 LLM 分析并给出改进建议（描述更清晰/添加约束/规范输出格式） |
| 自动生成 | 描述期望的输出效果，由 LLM 自动生成提示词草稿                 |
| 效果对比 | 原版本 vs 优化版本并排对话测试                         |

### 3.6 智能体配置（Agent Studio）

**功能目标**：提供完整的智能体配置能力，覆盖身份定义、模型选择、工具绑定、知识库接入、行为策略等所有维度。

> 详细设计见 `04-agent-studio.md`，本文档只做能力清单概览。

**完整配置维度：**

```
Agent 配置
├── 1. 基础信息
│   ├── 名称、描述
│   ├── 头像（图片上传 / Emoji）
│   ├── 标签（便于分类管理）
│   └── 开场白与建议问题
│
├── 2. 模型配置
│   ├── 主模型选择（从已启用模型列表选择）
│   ├── 推理参数：temperature / top_p / max_tokens
│   ├── 推理模型预算（thinking_budget，针对 o1/DeepSeek-R1 等）
│   ├── 备用模型（fallback，主模型不可用时自动切换）
│   └── 上下文窗口策略（截断 / 摘要压缩 / 滑动窗口）
│
├── 3. 提示词
│   ├── 系统提示词（含变量插值）
│   ├── 引用提示词模板库
│   └── 输出格式约束（TEXT / STRUCTURED + JSON Schema）
│
├── 4. 工具配置
│   ├── 工具列表（可选平台内置工具 / MCP 工具 / 自定义工具）
│   ├── 工具调用策略（auto / required / none）
│   └── 每个工具的参数默认值配置
│
├── 5. 知识库配置
│   ├── 关联知识库列表（可多选）
│   ├── 检索策略覆盖（可覆盖知识库默认配置）
│   └── 检索结果注入位置（system prompt / user message）
│
├── 6. Skill 配置
│   ├── 关联 Skill 列表（智能体可将 Skill 作为能力项注册）
│   └── Skill 调用时机（由 LLM 自主决定）
│
├── 7. 记忆配置
│   ├── 会话内记忆策略（截断 / 摘要 / 滑动窗口）
│   ├── 跨会话记忆开关（情节记忆）
│   └── 用户个性化记忆开关（记住用户偏好）
│
├── 8. 行为策略
│   ├── 最大循环步数（ReAct 循环上限，防止无限推理）
│   ├── 任务级超时（整体超时，非单次 LLM 调用）
│   └── Human-in-the-Loop 节点（需人工审批的操作类型）
│
└── 9. 安全与限制
    ├── 输入过滤规则（敏感词 / PII 脱敏 / Prompt Injection 检测）
    ├── 输出过滤规则（违规内容过滤 / 输出格式校验）
    ├── 调用频率限制（RPM / TPM 配额）
    └── 访问权限（谁可以调用此智能体）
```

### 3.9 多智能体编排（Multi-Agent Orchestration）

**功能目标**：支持多个智能体协作完成复杂任务，通过标准化能力声明实现灵活任务分配。

> 详细设计见 `06-workflow-multiagent.md`。

**编排模式：**

| 模式            | 描述                                  | 适用场景      | 优先级 |
| ------------- | ----------------------------------- | --------- | --- |
| Supervisor 模式 | Orchestrator 动态决定调用哪个子 Agent，汇总后再决策 | 开放性复杂任务   | P3  |
| Pipeline 模式   | Agent 链式调用，前一个输出作为后一个输入             | 确定步骤的流程任务 | P3  |
| Parallel 模式   | 多 Agent 并发执行，结果聚合                   | 独立子任务并发   | P3  |
| Debate 模式     | 多 Agent 对同一问题给出不同视角，Aggregator 仲裁   | 高风险决策     | P4  |

**A2A 协议兼容：**

- 预留 Agent Card 标准（Agent 能力声明文档）
- 跨平台 Agent 互联接口（HTTP + JSON-RPC）
- A2A 协议版本：兼容 Linux Foundation AAIF v1.0

### 3.13 可观测性（Observability）

**功能目标**：全链路追踪智能体的每一次运行，为排查问题、性能优化、成本管控提供数据支撑。

> 详细设计见 `08-observability-evaluation.md`。

**追踪层次：**

```
Session（会话）
└── Turn（一轮对话）
    └── AgentRun（一次 Agent 执行）
        ├── Plan（规划步骤，可选）
        ├── LLM Call（每次模型调用）
        │   ├── Input: messages + parameters
        │   ├── Output: content / tool_calls
        │   ├── Tokens: prompt_tokens + completion_tokens
        │   └── Latency: ms
        ├── Tool Call（每次工具调用）
        │   ├── Tool Name + Parameters
        │   ├── Response
        │   └── Latency: ms
        ├── Skill Call（每次 Skill 调用，类似 Tool）
        ├── Knowledge Retrieval（每次知识库检索）
        │   ├── Query
        │   ├── Retrieved Chunks（含相似度分数）
        │   └── Latency: ms
        └── Final Output
```

| 功能         | 描述                               | 优先级 |
| ---------- | -------------------------------- | --- |
| 调用量趋势      | 按天/小时的调用次数折线图                    | P0  |
| 成功率趋势      | 成功 / 失败 / 异常率                    | P0  |
| P95 延迟趋势   | 响应时间分布                           | P0  |
| Token 消耗统计 | 输入/输出 Token 分别统计，支持按模型/Agent/时间段 | P1  |
| 成本视图       | 按供应商定价折算，按智能体/时间段分摊              | P1  |
| 预算告警       | 设置 Token/成本预算，超阈值告警              | P2  |
| 会话日志       | 全量对话记录，支持全文检索（ES）                | P1  |
| 链路追踪视图     | 单次调用的完整 Trace 展开（树形可视化）          | P2  |
| 会话回放       | 完整对话 + 推理步骤时序回放                  | P3  |
| 异常分类       | 失败原因自动分类（超时/工具失败/模型拒绝/幻觉等）       | P1  |

### 3.15 安全与治理（Security & Guardrails）

**功能目标**：保护平台和用户不受恶意输入、数据泄露、滥用等安全威胁影响。

> 详细设计见 `08-observability-evaluation.md`。

**防护层次：**

```
输入层防护（Input Guardrails）
├── Prompt Injection 检测（防止用户注入恶意指令改变 Agent 行为）
├── 敏感词过滤（可配置词库）
├── PII 脱敏（身份证/手机号/邮箱等个人信息脱敏后再传给 LLM）
└── 输入长度限制

运行时防护（Runtime Guardrails）
├── 工具调用权限边界（Agent 只能调用被授权的工具）
├── 外部请求域名白名单（HTTP 工具只能访问白名单域名）
├── 最大循环步数限制
└── 任务级超时

输出层防护（Output Guardrails）
├── 违规内容过滤（基于规则 + LLM 判断）
├── 幻觉检测（输出中的关键信息来源可溯源性验证）
└── JSON Schema 强制校验（STRUCTURED 模式）

治理与配额
├── 租户级 TPM/RPM 配额（每分钟 Token / 请求数限制）
├── Agent 级调用频率限制
├── 安全审计日志（所有安全事件记录，与 admin-service 集成）
└── 黑名单用户管理（禁止特定用户调用智能体）
```
