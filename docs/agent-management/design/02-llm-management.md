## 2. 功能概述

大模型接入管理（LLM Management）是平台的底层基础设施能力，由**平台管理员**操作，为所有租户的智能体提供稳定、安全、可计量的 LLM 调用能力。

**核心职责：**
- 统一管理所有 LLM 供应商的接入配置
- 安全管理 API Key（不在响应或日志中暴露明文）
- 提供可用性验证，确保接入的模型可正常调用
- 控制模型对租户的可见范围

**功能范围（本文档覆盖）：**
- 模型列表管理（增删改查）
- 模型能力标注
- 多 API Key 池化管理
- 一键可用性测试
- 定时可用性拨测
- 模型可见性控制

---

## 3. 用户角色与权限

| 角色 | 权限 |
|------|------|
| 平台管理员 | 增删改查模型、管理密钥、设置模型可见性、执行可用性测试 |
| 租户管理员 | 查看本租户可见的已启用模型列表（只读）|
| 租户开发者 | 在智能体配置时选择启用的模型（只读查询）|

## 4. 模型列表管理

### 4.1 模型列表页

**页面功能：**
- 以表格形式展示所有已接入的大模型
- 支持多条件筛选（模型名称、模型标识、供应商类型、状态）
- 筛选条件变更后立即触发查询（无需点击查询按钮）
- 按创建时间降序排列，支持切换排序字段
- 分页展示（默认 20 条/页）

**表格列定义：**

| 列名 | 说明 | 宽度 |
|------|------|------|
| 模型名称 | 展示名称，默认模型旁显示「默认」徽标 | 200px |
| 模型标识 | 调用时的 model 字段（如 `gpt-4o`）| 180px |
| 供应商 | 供应商类型（OpenAI / Azure / Claude / 通义 / ...）| 130px |
| 可用性 | 最近测试结果：可用（绿）/ 存在问题（红）/ 未测试（灰）| 100px |
| 状态 | 启用 / 停用 | 80px |
| 可见范围 | 全局 / 部分租户 / 隐藏 | 100px |
| 最近测试时间 | 最后一次测试的时间 | 160px |
| 操作 | 一键测试 / 编辑 / 删除（固定右侧）| 180px |

**工具栏：**
- 搜索输入框（同时匹配模型名称和模型标识）
- 供应商类型下拉筛选
- 状态筛选（启用/停用/全部）
- 「新增模型」按钮（右侧）

### 4.2 新增模型

通过右侧 Drawer 展开表单，表单分 4 个区块：

#### 区块一：基础信息

| 字段 | 类型 | 必填 | 校验规则 | 说明 |

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 鉴权方式 | 当前固定为 API Key | — | 后续扩展 OAuth 等 |
| API Key | 密码输入框 | ✅ | 输入明文 → 提交后加密存储，不回显原文 |

**提交行为：**
- 新增时：明文 Key 在后端加密后存储，前端不缓存，不返回明文
- 编辑时：API Key 字段为空表示不修改，有值则更新

#### 区块四：模型能力标注（可选，供配置时参考）

| 字段 | 类型 | 说明 |

| 支持音频输入 | 开关 | 是否支持音频输入 |

| 输入 Token 单价（USD/1M）| 小数 | 用于成本统计，非必填 |
| 输出 Token 单价（USD/1M）| 小数 | 用于成本统计，非必填 |

**校验规则：**
- 模型标识在全平台唯一
- Base URL 格式校验（必须是有效 URL）
- 超时时间范围：1000ms - 300000ms

### 4.3 编辑模型

**可编辑字段（限制编辑以保证接入稳定性）：**

| 字段 | 可编辑 | 说明 |
|------|--------|------|
| 模型名称 | ✅ | 仅展示名称，不影响调用 |
| 启用状态 | ✅ | 随时可启用/停用 |
| 是否默认模型 | ✅ | 设为默认时自动取消其他模型 |
| 请求超时 | ✅ | 不影响接入身份 |
| 模型能力标注所有字段 | ✅ | 仅描述信息，不影响调用 |
| Token 单价 | ✅ | 仅计费信息 |
| API Key | ✅ | 可更新密钥，空值则保持不变 |

**不可编辑字段（变更需重新创建）：**
- 模型标识（核心调用身份）
- 供应商类型（决定使用哪个适配器）
- Base URL
- API 路径 / API 版本

> 如需修改不可编辑字段，应通过「新建模型」方式迁移，确保存量智能体调用不受影响。

### 4.4 删除模型

**删除前置校验：**
1. 查询有多少个智能体当前正在引用此模型（包括草稿态和已发布）
2. 若引用数 > 0，弹窗提示"该模型已被 N 个智能体使用，无法删除"，展示引用关系列表，引导到具体智能体修改
3. 若引用数 = 0，弹出二次确认对话框，确认后执行软删除（is_deleted = 1）

**删除结果反馈：**
- 成功：表格中移除该行，Toast 提示"删除成功"
- 失败（有引用）：弹窗展示引用关系

## 5. 一键可用性测试

### 5.1 测试流程

```
[平台管理员] 点击「一键测试」（行内按钮）
    │
    ▼
按钮变为 loading 状态（旋转图标 + 「测试中」文字）
    │
    ▼ POST /api/platform/agent/models/{id}/test
    │
    ▼ [agent-service]
    │   1. 从 DB 读取模型配置（Base URL / 接口路径 / providerType）
    │   2. 从 Secret Manager 获取 API Key（解密）
    │   3. 构建测试请求（发送固定测试 Prompt）
    │   4. 调用对应 LLM 适配器的 test() 方法
    │   5. 计时记录耗时（ms）
    │   6. 写回 DB：lastTestStatus / lastTestMessage / lastTestAt
    │
    ▼
返回测试结果（最大等待 min(timeoutMs, 30s)）
    │
    ├── 成功：按钮恢复，可用性 Badge 更新为「可用」（绿）
    └── 失败：按钮恢复，可用性 Badge 更新为「存在问题」（红）
         弹出 Toast 展示错误信息（错误类型 + 错误消息）
```

### 5.2 测试请求规范

**统一测试 Prompt（固定，不暴露给用户修改）：**
```json
{
  "messages": [
    {"role": "user", "content": "Hi, respond with 'ok' only."}
  ],
  "max_tokens": 10,
  "stream": false
}
```

**测试成功判定：**
- HTTP 状态码 200
- 响应体包含有效的 content 字段（非空）

**错误分类与用户友好提示：**

| 错误类型 | 触发条件 | 展示给用户的消息 |
|---------|---------|----------------|
| 认证失败 | HTTP 401 / 403 | "认证失败：API Key 无效或权限不足，请检查密钥配置" |
| 配额超限 | HTTP 429 | "配额超限：API 调用频率或用量已超限，请稍后重试" |
| 模型不存在 | HTTP 404 | "模型不存在：模型标识可能有误，请核查" |
| 请求超时 | 连接超时 / 读取超时 | "请求超时：LLM 供应商未在指定时间内响应，可适当增加超时配置" |
| 网络错误 | 连接失败 | "网络错误：无法连接到 LLM 供应商，请检查 Base URL 和网络访问" |
| 服务异常 | HTTP 5xx | "供应商服务异常：LLM 供应商返回 5xx 错误，请稍后重试" |
| 响应格式异常 | 无法解析响应 | "响应异常：供应商返回的数据格式不符合预期" |

### 5.3 可用性状态定义

| 状态 | 枚举值 | 展示样式 | 含义 |
|------|-------|---------|------|
| 未测试 | `UNTESTED` | 灰色标签 | 从未执行过测试 |
| 可用 | `AVAILABLE` | 绿色标签 | 最近一次测试成功 |
| 存在问题 | `ISSUE` | 红色标签 | 最近一次测试失败 |

## 6. 多 API Key 池化管理

> 优先级 P1，本节描述完整设计，MVP 阶段仅实现单 Key 关联。

### 6.1 功能背景

单个 API Key 通常有 RPM（每分钟请求数）和 TPM（每分钟 Token 数）限制。当平台并发调用量高时，单 Key 会遇到 429 限流。Key 池化通过多个 Key 轮转分摊请求压力。

### 6.2 Key 池配置（在模型配置中扩展）

每个模型配置可关联多个密钥（一对多关系）：

| 字段 | 说明 |
|------|------|
| 密钥 ID | 关联 Secret Manager 中的凭据 |
| 别名 | 方便管理的备注（如"OpenAI 主 Key"、"OpenAI 备用 Key 1"）|
| 权重 | 轮转权重（数字，默认 1），权重越高分配越多请求 |
| 状态 | 启用 / 停用 / 已失效（自动标记：连续失败 N 次后自动标记为失效）|
| 最近测试状态 | 支持对单个 Key 独立测试 |

### 6.3 轮转策略

| 策略 | 描述 | 适用场景 |
|------|------|---------|
| 加权轮询（Weighted Round Robin）| 按权重比例循环分配 | 主备 Key（主 Key 权重高）|
| 随机（Random）| 按权重随机选择 | 无状态场景 |
| 最少使用（Least Used）| 选择当前调用量最少的 Key | 精细化负载均衡 |

**失效 Key 处理：**
- 连续 3 次（可配置）调用返回 401/403，自动标记该 Key 为「已失效」
- 路由时跳过已失效 Key
- 向平台管理员发送告警通知
- 管理员可手动将 Key 重置为「启用」状态

## 7. 定时可用性拨测

> 优先级 P2。

### 7.1 功能描述

平台自动定时执行所有「启用」状态模型的可用性测试，持续监控 LLM 供应商的服务状态。

### 7.2 配置

| 参数 | 默认值 | 可配置范围 | 说明 |
|------|-------|---------|------|
| 拨测间隔 | 10 分钟 | 5 分钟 - 60 分钟 | 定时执行频率 |
| 告警阈值 | 连续失败 2 次 | 1 - 10 次 | 触发告警的连续失败次数 |
| 拨测范围 | 所有启用模型 | 可选特定模型列表 | 默认测试全部 |

### 7.3 告警通知

拨测失败触发告警时：
- 平台管理员收到站内通知：「模型 [X] 可用性测试失败（连续 N 次），最后错误：[错误信息]」
- 可扩展：发送邮件 / Webhook 通知（P3）

## 8. 模型可见性控制

### 8.1 可见范围配置

在模型配置中设置：

| 可见性选项 | 枚举值 | 效果 |
|----------|-------|------|
| 全局可见 | `PUBLIC` | 所有租户均可在智能体配置中选用 |
| 指定租户 | `TENANT_WHITELIST` | 仅白名单内的租户可见，其他租户不展示 |
| 平台私有 | `PRIVATE` | 仅平台管理员可见，不对任何租户开放 |

### 8.2 租户端模型列表规则

租户开发者在智能体配置页选择模型时，查询 API 自动过滤：
- 仅返回状态为「启用」的模型
- 仅返回对当前租户可见的模型（根据可见性规则）
- 按模型名称排序，默认模型排在首位

## 9. 页面交互详细设计

### 9.1 列表页交互

```
加载完成后展示表格
    │
    ├── 搜索/筛选变更 → 实时触发查询（debounce 300ms）
    │
    ├── 点击「一键测试」
    │   ├── 行内按钮变为 loading（旋转图标）
    │   ├── 禁用该行其他操作按钮（防重复点击）
    │   ├── 等待响应（最长 30s）
    │   └── 结果返回 → 更新可用性 Badge → 恢复按钮
    │
    ├── 点击「编辑」→ 打开右侧 Drawer（预填当前数据，API Key 字段为空）
    │
    └── 点击「删除」
        ├── 发起引用数查询
        ├── 引用数 > 0 → 弹窗展示引用列表，禁止删除
        └── 引用数 = 0 → 二次确认弹窗 → 确认后执行删除 → 表格刷新
```

### 9.2 新增/编辑 Drawer 交互

```
Drawer 打开
    │
    ├── 选择供应商类型后
    │   └── 自动预填 Base URL（如果当前 Base URL 为空）
    │
    ├── 填写模型标识后（失焦时）
    │   └── 如有预置能力信息，提示「检测到 GPT-4o，是否自动填写能力标注？」
    │
    ├── API Key 字段
    │   ├── 新增：必填，加密存储
    │   └── 编辑：选填，留空表示不修改，有值则更新
    │
    ├── 表单提交
    │   ├── 前端校验（必填项/格式/唯一性预检）
    │   ├── 提交成功 → 关闭 Drawer → 列表刷新 → Toast 提示
    │   └── 提交失败 → 保持 Drawer 开启 → 展示错误信息
    │
    └── 关闭 Drawer → 提示「有未保存的修改，确认关闭？」（仅在有修改时）
```

## 11. 接口说明（概要）

> 详细接口定义见 `10-api-spec.md`。

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/platform/agent/models` | 分页查询模型列表（含筛选）|
| POST | `/api/platform/agent/models` | 新增模型 |
| GET | `/api/platform/agent/models/{id}` | 查询模型详情 |
| PUT | `/api/platform/agent/models/{id}` | 编辑模型（仅允许编辑字段）|
