## 1. 功能概述

用户中心模块新增用户导入功能，支持通过Excel文件批量导入用户数据，单次最多支持2万条数据导入。

## 2. API 设计

### 2.1 模板下载接口

**接口路径**: `GET /api/v1/app-users/import-template`

**响应**: Excel模板文件下载

**权限**: 无需权限（公开接口）

### 2.2 数据校验接口

**接口路径**: `POST /api/v1/app-users/import-validate`

**请求方式**: `multipart/form-data`

**请求参数**:

| 参数名  | 类型            | 必填 | 说明             |
| ---- | ------------- | -- | -------------- |
| file | MultipartFile | 是  | Excel文件(.xlsx) |

**响应结构**:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "taskId": "import_20240101_123456",
    "total": 20000,
    "validCount": 19950,
    "invalidCount": 50,
    "canProceed": true
  }
}
```

### 2.3 校验结果下载接口

**接口路径**: `GET /api/v1/app-users/import-validate-result/{taskId}`

**响应**: Excel文件下载，包含所有数据和校验结果（错误行新增"错误说明"列）

**权限**: `app:user:import`

### 2.4 执行导入接口

**接口路径**: `POST /api/v1/app-users/import-execute`

**请求方式**: `application/json`

**请求参数**:

| 参数名    | 类型     | 必填 | 说明     |
| ------ | ------ | -- | ------ |
| taskId | String | 是  | 校验任务ID |

**响应结构**:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "importTaskId": "import_exec_20240101_123456"
  }
}
```

### 2.5 导入进度查询接口

**接口路径**: `GET /api/v1/app-users/import-progress/{importTaskId}`

**响应结构**:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "status": "PROCESSING",
    "total": 19950,
    "processed": 10000,
    "success": 9980,
    "failed": 20,
    "progress": 50,
    "errors": [
      {
        "row": 103,
        "mobile": "13900139000",
        "reason": "插入时手机号已被其他用户占用"
      }
    ]
  }
}
```

**status 枚举值**:

* `PENDING` - 等待处理

* `PROCESSING` - 处理中

* `COMPLETED` - 已完成

* `FAILED` - 失败

### 3.1 导入模板

| 列号 | 字段名 | 说明           | 必填 | 示例                     |
| -- | --- | ------------ | -- | ---------------------- |
| A  | 昵称  | 用户昵称         | 否  | 张三                     |
| B  | 手机号 | 用户手机号        | 是  | 13800138000            |
| C  | 邮箱  | 用户邮箱         | 否  | <zhangsan@example.com> |
| D  | 性别  | 男/女/未知       | 否  | 男                      |
| E  | 生日  | yyyy-MM-dd格式 | 否  | 1990-01-01             |
| F  | 状态  | 正常/禁用        | 否  | 正常                     |
| G  | 标签  | 多个标签用逗号分隔    | 否  | VIP,活跃用户               |

### 3.2 校验结果文件

在原模板基础上新增一列：

| 列号 | 字段名  | 说明               |
| -- | ---- | ---------------- |
| H  | 错误说明 | 校验错误的原因说明，正确数据为空 |

## 4. 校验规则

| 字段  | 校验规则                            |
| --- | ------------------------------- |
| 手机号 | 必填，11位数字，格式校验，唯一性校验             |
| 邮箱  | 邮箱格式校验（如填写），唯一性校验（如填写）          |
| 昵称  | 长度限制2-20字符                      |
| 性别  | 枚举值：男/女/未知，默认"未知"               |
| 生日  | 日期格式校验，不能晚于当前日期                 |
| 状态  | 枚举值：正常/禁用，默认"正常"                |
| 标签  | 标签必须已存在                         |
| 文件  | 非空校验，格式校验（.xlsx），数据量限制（≤20000条） |

## 5. 导入逻辑流程

### 5.1 整体流程

```
┌─────────────────────────────────────────────────────────────────┐
│                         用户导入流程                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 上传Excel文件                                                │
│     ↓                                                           │
│  2. 调用校验接口 ──────────────────────────────────────────────→ │
│     ↓                                                           │
│  3. 后端校验数据，生成taskId，缓存校验结果                         │
│     ↓                                                           │
│  4. 返回校验结果（总数、正确数、错误数）                            │
│     ↓                                                           │
│  5. 前端展示校验结果                                              │
│     ├─ 有错误 → 可下载错误报告Excel（含错误说明列）                 │
│     └─ 无错误或用户确认 → 执行导入                                 │
│     ↓                                                           │
│  6. 调用执行导入接口                                              │
│     ↓                                                           │
│  7. 后端异步执行导入，返回importTaskId                             │
│     ↓                                                           │
│  8. 前端轮询进度接口，显示进度条                                    │
│     ↓                                                           │
│  9. 导入完成，显示最终结果                                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 5.2 校验流程

```
1. 接收Excel文件
   ↓
2. 校验文件格式（.xlsx）
   ↓
3. 校验文件非空
   ↓
4. 校验数据量（≤20000条）
   ↓
5. 解析Excel数据
   ↓
6. 逐行校验数据
   ├─ 格式校验
   ├─ 业务校验（唯一性、标签存在性）
   └─ 记录错误信息到每行数据
   ↓
7. 生成taskId，缓存校验结果（Redis，有效期30分钟）
   ↓
8. 返回校验统计结果
```

### 5.3 导入执行流程

```
1. 接收taskId
   ↓
2. 从缓存获取校验通过的数据
   ↓
3. 创建异步导入任务，生成importTaskId
   ↓
4. 返回importTaskId给前端
   ↓
5. 异步执行导入（后台线程）
   ├─ 分批处理（每批500条）
   ├─ 更新进度到缓存
   ├─ 批量插入数据库
   └─ 记录插入失败的行
   ↓
6. 导入完成，更新最终状态
```

## 6. 技术实现方案

### 6.1 后端实现

#### 6.1.1 环境配置

* **Docker环境**：添加字体支持（fontconfig ttf-dejavu）

* **MyBatis Plus**：配置MetaObjectHandler自动填充时间字段

#### 6.1.2 新增依赖

项目已包含 EasyExcel 依赖，无需额外添加。

#### 6.1.3 新增类文件

**DTO/VO类**:

* `UserImportDTO.java` - Excel行数据映射对象

* `UserImportValidateResultVO.java` - 校验结果返回对象

* `UserImportProgressVO.java` - 导入进度返回对象

**缓存类**:

* `ImportCacheService.java` - 导入任务缓存服务（Redis）

**异步处理类**:

* `ImportTaskExecutor.java` - 异步导入任务执行器

**监听器**:

* `HeaderValidationListener.java` - EasyExcel表头校验监听器

**配置类**:

* `MyMetaObjectHandler.java` - MyBatis Plus自动填充处理器

#### 6.1.5 Redis缓存设计

```
# 校验结果缓存
Key: import:validate:{taskId}

```

## 8. 注意事项

### 8.1 环境依赖

* **Docker容器**：必须安装字体包（fontconfig ttf-dejavu）

* **Redis**：用于缓存校验结果和导入进度

* **JDK**：17+

### 8.2 性能考虑

* 单次导入最多支持20000条数据

* 校验结果缓存30分钟，超时需重新上传

* 分批插入数据库（每批500条），避免单次事务过大

* 异步执行导入，避免阻塞请求

* 文件大小限制（最大10MB），防止大文件导致系统负载过高

* 表头快速校验，避免解析整个文件后才发现表头错误

### 8.3 事务处理

* 校验阶段不写入数据库

* 导入阶段分批提交事务

* 单批失败不影响已成功批次

* 记录每批次的失败详情

### 8.4 并发控制

* 同一管理员同时只能有一个导入任务

* 前端在校验/导入过程中禁用导入按钮

* 后端通过Redis锁控制并发

### 8.5 日志记录

* 记录导入操作日志

* 包含操作人、导入数量、成功/失败数量

* 记录导入任务ID便于追溯

### 8.6 错误处理

* **Excel校验异常**：统一捕获所有类型的异常，返回友好的错误消息

* **文件格式错误**：提供清晰的错误提示，引导用户使用正确的模板

* **前端错误处理**：对后端返回的null值进行安全处理，避免前端崩溃

* **异常日志**：详细记录异常信息，便于问题排查

### 8.7 缓存清理

* 校验结果30分钟后自动过期

* 导入进度1小时后自动过期

* 导入完成后可手动清理缓存
